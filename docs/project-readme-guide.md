# Project README Guide

Each project should have a clear README.

## Structure

### Title
What the project is.

### Goal
What problem you solve.

### Dataset
What data you use.

### Approach
What models or methods you tried.

### Results
What happened.

### Lessons Learned
What you discovered.

### Next Steps
What you will improve.

## Example

Bad:
- "trained model"

Good:
- "Random Forest improved RMSE from 0.82 to 0.65 after feature scaling"

## Principle
Your README is as important as your code.
